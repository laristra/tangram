45 /*
46  @namespace Tangram
47  The Tangram namespace houses all of the code within Tangram.
48 
49  Cells (aka zones/elements) are the highest dimension entities in a mesh
50  Nodes (aka vertices) are lowest dimension entities in a mesh
51  Faces in a 3D mesh are 2D entities, in a 2D mesh are 1D entities
52  BOUNDARY_FACE is a special type of entity that is need so that process
53  kernels can define composite vectors (see src/data_structures) on
54  exterior boundary faces of the mesh only
55 
56  Wedges are special subcell entities that are a simplicial
57  decomposition of cell. In 3D, a wedge is tetrahedron formed by one
58  point of the edge, the midpoint of the edge, the "center" of the
59  face and the "center" of the cell volume. In 2D, a wedge is a
60  triangle formed by an end-point of the edge, the mid-point of the
61  edge and the center of the cell. In 1D, wedges are lines, that are
62  formed by the endpoint of the cell and the midpoint of the
63  cell. There are two wedges associated with an edge of cell face in
64  3D. The two wedges associated with an edge and face pair form a larger
65  simplex called a side
66 
67  Corners are also subcell entities that are associated uniquely with
68  a node of a cell. Each corner is the union of all the wedges incident
69  upon that node in the cell
70 
71  Facets are the boundary entity between two wedges in adjacent
72  cells. In 3D, a facet is a triangular subface of the cell face
73  shared by two wedges in adjacent cells. In 2D, a facet is half of
74  an edge that is shared by two wedges in adjacent cells
75  */

205 /* In the context of interface reconstruction methods based on the nested
206  dissections algorithm, the iterative methods are used to find the
207  position of the cutting plane.
208  An example of iterative methods dealing with volumes is solving for
209  the cutting distance: given a plane's orientation, find its position
210  such that the volume of a part of a polyhedron in a given half-space
211  matches the target volume. arg_eps will correspond to a negligible
212  change in the cutting distance, and fun_eps will correspond to a
213  negligible discrepancy in volume.
214  An example of iterative methods dealing with centroids is finding the
215  cutting plane's orientation: find such a normal that the distance between
216  the centroid of a part of a polyhedron in a given half-space
217  is the closest to the given reference centroid. arg_eps will correspond
218  to a negligible change in the orientation, and fun_eps will correspond to a
219  negligible distance between centroids.
220  Note that distance and volume tolerances for the cutting distance calculation
221  algorithm, which is used by all the implemented interface reconstruction methods,
222  are also used to eliminate degenerate vertices (i.e. vertices with distance below
223  distance tolerance) and materials (i.e. materials with volume below volume tolerance).
224 */
225 struct IterativeMethodTolerances_t {
226  int max_num_iter; // Max number of iterations
227  double arg_eps; // Tolerance on the arguments of the function
228  double fun_eps; // Tolerance on the value of the function
229 };

243 template <int D>
244 inline bool overlapping_boxes(const BoundingBox_t<D>& bb1,
245  const BoundingBox_t<D>& bb2,
246  double dst_eps) {
247  for (int idim = 0; idim < D; idim++)
248  if ((bb1.min[idim] > bb2.max[idim] - dst_eps) ||
249  (bb1.max[idim] < bb2.min[idim] + dst_eps))
250  return false;
251 
252  return true;
253 }

270 template <typename T>
271 std::vector<int> sorted_indices(const std::vector<T> &v,
272  bool ascending = true) {
273  std::vector<int> ids(v.size());
274  std::iota(ids.begin(), ids.end(), 0);
275 
276  // Sort indices based on values of v
277  if (ascending)
278  std::sort(ids.begin(), ids.end(),
279  [&v](int i1, int i2) { return v[i1] < v[i2]; });
280  else
281  std::sort(ids.begin(), ids.end(),
282  [&v](int i1, int i2) { return v[i1] > v[i2]; });
283 
284  return ids;
285 }
